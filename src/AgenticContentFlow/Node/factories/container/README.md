# Container Node Factory System

This document provides a comprehensive guide to the new Container Node Factory System, which works alongside the existing Cell Node Factory to provide JSON-driven container node creation.

## Overview

The Container Node Factory System creates factory-generated container nodes (DataNode, PageNode, StatisticsNode, InvisibleNode) from JSON configurations, providing:

- **Expand/Collapse functionality** with automatic child visibility management
- **Depth-based coloring** with multiple color schemes
- **Resizing capabilities** with corner handles
- **Parent-child relationships** for organizing content
- **Custom styling** per container type
- **Menu actions** specific to each container category

## Architecture Comparison

### Cell Factory (Process/Preview Nodes)
- **Purpose**: Business logic and data transformation
- **Appearance**: Square, uniform styling
- **Behavior**: Process execution, data flow
- **Examples**: RestNode, LogicalNode, ContentNode

### Container Factory (Container Nodes)
- **Purpose**: Organization and layout
- **Appearance**: Expandable, depth-colored styling
- **Behavior**: Expand/collapse, contain children
- **Examples**: DataNode, PageNode, StatisticsNode, InvisibleNode

## Quick Start

### 1. Initialize Both Factory Systems

```typescript
import { initializeAllNodeFactories } from './Node/factories';

// During application startup
await initializeAllNodeFactories();
```

### 2. Use Container Nodes in Flows

```typescript
// Example container node instances
const containerNodes = [
  {
    id: "data-source",
    type: "datanode", // Generated by container factory
    data: {
      label: "User Data",
      dataFormat: "json",
      records: 1500,
      expanded: false
    },
    position: { x: 100, y: 100 }
  },
  {
    id: "content-page",
    type: "pagenode", // Generated by container factory
    data: {
      label: "Landing Page",
      pageType: "html",
      expanded: true
    },
    position: { x: 400, y: 100 }
  }
];
```

## Container Node Types

### DataNode
- **Visual**: File/folder appearance with shadow
- **Behavior**: Expandable, resizable, can contain data sources
- **Handles**: Data output (right), control flow (top/bottom), references (left)
- **Menu**: Load Data, Open File, Download, Share

### PageNode  
- **Visual**: Standard container with clean styling
- **Behavior**: Expandable, resizable, can contain page content
- **Handles**: Control flow (top/bottom), data input (left), analytics output (right)
- **Menu**: Edit HTML, Preview Page

### StatisticsNode
- **Visual**: Analytics-focused with chart styling
- **Behavior**: Expandable, resizable, does not contain children
- **Handles**: Analytics input (left)
- **Menu**: Export Data, Refresh Metrics, Configure Charts

### InvisibleNode
- **Visual**: Transparent when expanded, purple when collapsed
- **Behavior**: Expandable, resizable, layout container
- **Handles**: Control flow (top/bottom)
- **Menu**: Toggle Container Visibility

```typescript
// Automatic color schemes based on node depth
const colorSchemes = {
  default: ['#f3f4f6', '#e5e7eb', '#d1d5db', '#9ca3af'],
  purple: ['#faf5ff', '#f3e8ff', '#e9d5ff', '#c4b5fd'],

  green: ['#f0fdf4', '#dcfce7', '#bbf7d0', '#86efac']

```

### 2. Expand/Collapse State Management

```typescript
// Automatic child visibility management
const expandedState = {
  expanded: true,
  childCount: 3,
  dimensions: {
    current: { width: 300, height: 300 },
    collapsed: { width: 300, height: 200 },
    expanded: { width: 300, height: 300 }
  }
};
```

### 3. Custom Content Generation

```typescript
// Category-specific content in expanded state
switch (config.category) {
  case 'data':
    return <DataSourceInfo records={nodeData.records} />;
  case 'page':
    return <PageContentInfo htmlLength={nodeData.htmlContent?.length} />;
  case 'statistics':
    return <AnalyticsOverview metrics={nodeData.metrics} />;
}
```

## Integration Benefits

1. **Unified Factory System**: Both cell and container nodes use JSON-driven configuration
2. **Consistent API**: Same initialization and registration patterns
3. **Type Safety**: Full TypeScript support with proper type definitions
4. **Extensibility**: Easy to add new container types via JSON
5. **Maintainability**: Centralized styling and behavior management
6. **Performance**: Efficient expand/collapse with automatic child management

## Migration Path

### For Existing Container Nodes

1. **Keep existing nodes**: Current DataNode, PageNode, etc. continue to work
2. **Optional upgrade**: Gradually migrate to factory-generated versions
3. **Configuration-driven**: Replace hardcoded logic with JSON configurations
4. **Enhanced features**: Gain automatic depth coloring, improved expand/collapse

### For New Container Nodes

1. **Use factory system**: Create JSON configurations instead of manual components
2. **Leverage built-ins**: Extend existing container configurations
3. **Custom content**: Add category-specific content generation
4. **Consistent styling**: Automatic depth-based coloring and container styling
